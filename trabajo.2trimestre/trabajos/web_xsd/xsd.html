Resumen de XML Schema

La validación mediante XML Schema es fundamental porque asegura que los datos sigan una estructura y tipos específicos, garantizando la integridad de la información al intercambiar datos entre diferentes sistemas.

1. ¿Que es el Esquema?

  • Usa extension .xsd.
  • Usa espacios de nombres (namespaces).
  • Sirve para definir el orden de elementos, si son obligatorios y sus valores fijos.
  • La raiz siempre es <xs:schema>.

2. Tipos de Datos

XSD ya trae tipos listos para usar:

  • string: Texto normal.
  • integer / decimal: Numeros.
  • boolean: true o false.
  • date / time: Fechas y horas.

3. Elementos Simples vs Complejos

Segun las diapositivas, tenemos dos formas de agrupar:

  • Simples: Solo contienen texto. Se definen con type="...".
  • Complejos: Contienen otros elementos o atributos. Usan <xs:complexType> y <xs:sequence> (para orden fijo) o <xs:choice> (para elegir uno).
  • Referencias: Se usa ref para organizar mejor el documento y no repetir codigo.

4. Restricciones y Atributos

Podemos limitar los datos con "facets" dentro de un simpleType:

  • MinInclusive / MaxInclusive: Para rangos de numeros (ej: del 1 al 10).
  • Enumeration: Para crear una lista de valores permitidos (ej: rojo, verde, azul).
  • Pattern: Para usar expresiones regulares (formatos especificos).
  • Atributos: Se definen con <xs:attribute> y pueden ser opcionales o obligatorios (required).
